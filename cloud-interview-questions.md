# ☁️ Cloud Computing – Interview Questions & Answers

---

### 1. What is cloud computing?
Cloud computing is the on-demand delivery of IT resources over the internet with pay-as-you-go pricing instead of owning physical data centers.

---

### 2. What are the types of cloud services?
- **IaaS** – Infrastructure as a Service (e.g., AWS EC2)
- **PaaS** – Platform as a Service (e.g., AWS Elastic Beanstalk)
- **SaaS** – Software as a Service (e.g., Google Workspace)

---

### 3. What is the difference between scalability and elasticity?
- **Scalability** – Manually increasing resources to handle growth
- **Elasticity** – Automatically scaling resources up/down based on demand in real-time

---

### 4. What is the AWS Shared Responsibility Model?
- **AWS** is responsible for security **of** the cloud (hardware, infrastructure)
- **Customer** is responsible for security **in** the cloud (data, apps, IAM)

---

### 5. What is a VPC?
A Virtual Private Cloud (VPC) is a logically isolated network in AWS where you can launch resources in a defined virtual network with full control over IP ranges, subnets, and routing.

---

### 6. What is the difference between S3 and EBS?
| Feature | S3 | EBS |
|---------|-----|-----|
| Type | Object storage | Block storage |
| Access | Via internet/API | Attached to EC2 |
| Use case | Files, backups, static assets | OS disks, databases |

---

### 7. What is serverless computing?
Serverless means you run code without provisioning or managing servers. AWS Lambda is the primary serverless compute service — you only pay for the time your code runs.

---

### 8. What is a CDN and which AWS service provides it?
A Content Delivery Network (CDN) caches content at edge locations worldwide to reduce latency. AWS CloudFront is the CDN service.

---

### 9. What is IAM in AWS?
Identity and Access Management (IAM) lets you control who can access AWS resources and what actions they can perform using users, groups, roles, and policies.

---

### 10. What are Availability Zones (AZs)?
Availability Zones are isolated data center locations within an AWS Region. Deploying across multiple AZs ensures high availability and fault tolerance.

---

> ✅ These questions are commonly asked in cloud, DevOps, and solutions architect interviews.
